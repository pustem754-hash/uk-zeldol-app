## 🔜 МОДУЛЬ 02: Auth SMS

**Цель:** Авторизация жителей через SMS код

**Основные задачи:**
1. Интеграция SMS шлюза (SMSC.ru или SMS.ru)
2. API endpoint для отправки кода (`/api/auth/send-code`)
3. API endpoint для проверки кода (`/api/auth/verify-code`)
4. Страница входа (`/login`)
5. Страница подтверждения кода (`/verify`)
6. Проверка номера в базе (защита от посторонних)
7. Создание сессии через Supabase Auth

**Технические детали:**
- SMS коды хранятся в таблице `sms_codes`
- Время жизни кода: 5 минут
- Rate limiting: не более 3 попыток в минуту
- После верификации создается профиль в `profiles`

**Промпт для Cursor:**
```
Создай систему авторизации через SMS для приложения УК:
1. API route для отправки 4-значного SMS кода на телефон
2. Проверку что номер есть в таблице profiles
3. API route для проверки введенного кода
4. Страницу /login с полем для телефона
5. Страницу /verify с полями для кода
6. Использовать Supabase Auth для создания сессии
7. Redirect на /dashboard после успешного входа
```

## 🔜 МОДУЛЬ 03: User Profile

**Цель:** Профиль жителя с множественными лицевыми счетами

**Основные задачи:**
1. Страница профиля (`/profile`)
