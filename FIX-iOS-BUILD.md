# Исправление проблемы с запуском iOS приложения

## Что было исправлено:

### 1. ✅ Конфигурация `eas.json`
- Убрано `simulator: true` из профиля `preview`
- Добавлено `buildConfiguration: "Release"` для production сборки
- Добавлен `resourceClass: "m-medium"` для оптимизации сборки

### 2. ✅ Конфигурация `app.json`
- Добавлен `scheme: "greenvalley"` для deep linking

## Что нужно сделать:

### Шаг 1: Пересобрать приложение

```bash
eas build --platform ios --profile preview --clear-cache
```

Флаг `--clear-cache` очистит кеш и пересоберет приложение с новыми настройками.

### Шаг 2: Установить новую сборку на iPhone

После завершения сборки:
1. Скачайте новый IPA файл
2. Или отсканируйте новый QR-код
3. Установите на iPhone

### Шаг 3: Если проблема сохраняется

Проверьте:

1. **Удалите старое приложение** с iPhone перед установкой новой версии

2. **Проверьте сертификаты**:
   ```bash
   eas credentials
   ```

3. **Попробуйте production профиль**:
   ```bash
   eas build --platform ios --profile production
   ```

4. **Проверьте логи устройства**:
   - Подключите iPhone к Mac
   - Откройте Console.app
   - Найдите ошибки при запуске приложения

## Возможные причины проблемы:

1. ❌ Старая сборка с неправильными настройками
2. ❌ Конфликт сертификатов
3. ❌ Проблемы с bundle identifier
4. ❌ Отсутствие необходимых разрешений

## Решение:

После пересборки с исправленными настройками приложение должно запускаться корректно.

Если проблема сохраняется, выполните:

```bash
# Полная очистка и пересборка
eas build --platform ios --profile preview --clear-cache --non-interactive
```
